01

组件调用顺序

存在一个默认管线，但管线会根据实时条件被跳过、重复、或倒回
评估
启动缓冲
力度压缩
语义载体
降级包装
出口策略
0评估阶段（不发声）
大脑评估三个变量，决定后续管线配置
社交风险
说了之后关系会受损吗？
0=安全 → 10=翻脸
确信度
我有多确定这件事？
0=直觉 → 10=铁证
紧迫度
必须现在说吗？
0=随时 → 10=来不及了
1启动阶段
社交风险 ≥ 4 → 放缓冲 ｜ 社交风险 < 4 → 跳过
なんかあのさていうかえっといやまあ
2压缩阶段
社交风险 ≥ 5 且 紧迫度 < 7 → 压缩 ｜ 否则跳过
ちょっと別になんとなくたぶん一応
3语义阶段
确信度越高 → 越重的词 ｜ 社交风险越高 → 越轻的词 ｜ 冲突时社交风险优先
違うおかしい微妙悪くない無理好き嫌
4包装阶段
确信度 ≤ 7 或 社交风险 ≥ 5 → 降级 ｜ 确信度 > 7 且 社交风险 < 3 → 跳过
気がするかもかなっぽい感じなんとも言えない
5出口阶段
想让对方接 → んだけど ｜ 想要共识 → よね ｜ 想断言 → でしょ ｜ 想逃 → ...沉默
んだけどよねじゃない？けどし...知らんけど

管线跳过规则

紧迫度 ≥ 9（紧急情况）
跳过阶段1-2-4，直接：语义 → 出口
危ない！→ 没有缓冲、没有压缩、没有降级
社交风险 = 0（自言自语）
跳过阶段1-2，可能跳过5
あー疲れた → 没有缓冲、没有压缩、没有出口策略
社交风险 ≥ 8（极度敏感话题）
阶段1-2-4全部加载，可能重复加载
なんかちょっと...なんていうか...違う感じが...しないでもない...んだけど → 缓冲×2 压缩×1 降级×2
确信度 = 10 且 社交风险低
跳过阶段1-2-4
それ間違ってるよ → 直接语义 + 简单出口

管线倒回规则

说到语义阶段时发现风险比预估高
倒回到启动阶段重来
違う...あ、いやなんか、ちょっと違う気がするんだけど → 先输出了语义，发现太直接，倒回加缓冲+压缩+降级
看到对方表情变化
中断当前管线，启动修正管线
それおかしく...あ、いやごめん、そういう意味じゃなくて
02

选择机制

同类组件之间如何选择——不是语法问题，是心理+社交的实时计算

缓冲组件选择

なんか
选择条件社交风险中等(4-6) + 内容不太重要 + 想降低存在感
心理状态「我说的可能不重要，别太当真」
不选的情况内容重要且紧急 / 完全安全的环境
あのさ
选择条件有明确的话要说 + 需要对方注意力 + 关系亲密
心理状态「我要讲一个事，你听着」
不选的情况不确定要不要说 / 对方是上级
ていうか
选择条件要修正前面的话 + 不想正面对撞
心理状态「你说的不是重点，我的才是」
不选的情况没有前文可修正 / 第一次开口
いや
选择条件第一反应是否定 + 来不及包装
心理状态「等等，不对」
不选的情况同意对方的时候 / 需要温和开场
まあ
选择条件不想深入 + 想快速给表面结论
心理状态「算了，差不多就这样吧」
不选的情况有强烈意见 / 事情很重要

降级方式选择：为什么有时直接说「違う」有时说「違う気がする」

直接说「違う」
条件确信度 ≥ 8 ｜ 社交风险 ≤ 3 ｜ 关系亲密或地位高 ｜ 紧迫度高
「違う気がする」
条件确信度 ≤ 7 或 社交风险 ≥ 5 ｜ 不想承担断言后果 ｜ 给自己留退路
「違うかも」
条件确信度 5-7 ｜ 社交风险中等 ｜ 比気がする更随意更短
「違うっぽい」
条件基于表面印象 ｜ 没认真想 ｜ 不想负任何责任
「なんとも言えない」
条件有明确判断但选择不说 ｜ 社交风险 ≥ 7 ｜ 后果不可控
降级程度 = 社交风险 × (1 - 亲密度) × (1 / 紧迫度)

出口选择

んだけど
效果打开通道不关门 — 想让对方推断你的意思
よね
效果把个人判断变成共享判断 — 需要对方确认
じゃない？
效果形式是问句，实际是断言 — 比よね更强势
...（沉默）
效果最高级的间接表达 — 什么都不说让对方自己填
知らんけど
效果事后免责声明 — 说了判断但撤回全部责任
03

修正机制

说话者在说的过程中实时监听自己、观察对方、调整输出
触发1：自检 — 说太直接了
说出核心判断后，内心评估：「刚才那句太重了」
それ間違ってる。
それ間違ってる...っていうか、なんか違う気がするんだけど
语义(間違ってる) → 自检(太重了) → 切换(っていうか) → 缓冲(なんか) → 替换语义(違う) → 降级(気がする) → 出口(んだけど)
触发2：对方反应 — 看到对方表情变了
对方皱眉/沉默/语气变冷
それおかしいでしょ。
それおかしい...あ、ごめん、言い方悪かった。なんかちょっと引っかかるところがあるんだけど
语义(おかしい) → 检测对方反应 → 中断 → 道歉(ごめん) → 认错(言い方悪かった) → 重启：缓冲(なんか) → 压缩(ちょっと) → 替换语义(引っかかる) → 出口(んだけど)
触发3：精度 — 自己说的不准确
说完后觉得「不对，我要表达的不是这个」
嫌いなんだよね。
嫌いっていうか...なんていうか、合わないのかな。嫌いじゃないんだけど、なんか...ね
语义(嫌い) → 自检(不准确) → 切换(っていうか) → 搜索(なんていうか) → 替换语义(合わない) → 降级(かな) → 再自检 → 否定前文(嫌いじゃない) → 出口(んだけど) → 留白(なんか...ね)
触发4：过度 — 对方把你的话放大了
对方的反应比预期更大：「え、辞めたいの？」
ちょっと疲れた
いやいや、そこまで言ってないって。ちょっと今日疲れただけ
检测过度解读 → 否定(いやいや) → 范围纠正(そこまで言ってない) → 重新限定(ちょっと今日) → 语义重述(疲れただけ)

修正标记选择

いや
最快的中断 — 反射性否定前文
ていうか
方向修正 —「不是那个，是这个」
そうじゃなくて
理解修正 —「你听错了」
ごめん
关系修复 — 先道歉再重说
あ、やっぱ
决定修正 —「我改主意了」
今のなしで
完全撤回 —「当我没说」

完整生成示例

场景：觉得对方说错了，但不想伤人
对方说：「あの店まずいよね」
你的内心：不对吧...我觉得那家店挺好吃的
STEP 0 — 评估（不发声）
大脑计算三个变量
社交风险 = 6
直接反驳朋友会让气氛尴尬
确信度 = 7
去过好几次，确实觉得好吃
紧迫度 = 2
不说也行，但想说
决定：说，但需要完整的包装管线
STEP 1 — 启动阶段
社交风险=6 ≥ 4 → 需要缓冲
なんか可以 — 降低分量
いや可以但偏硬 — 直接否定了对方
えー太惊讶 — 暗示对方很离谱
选择：えー（轻度惊讶）+ いや（轻度否定）——表达「跟我想的不一样」但不攻击
えー、いや...
STEP 2 — 压缩阶段
社交风险=6 ≥ 5 且 紧迫度=2 < 7 → 需要压缩
ちょっと标准选择
そんなに「没你说的那么差」比直接反驳温和
別に太冷淡
选择：そんなに — 不说「不一样」而是说「没那么严重」
えー、いや、そんなに...
STEP 3 — 语义阶段
确信度=7 但 社交风险=6 → 选轻量语义词
まずくない双重否定 =「不难吃」（实际=好吃）
美味しい太直接 — 正面否定了朋友
悪くない低温正面 — 但跟食物不够自然
选择：まずくなくない？ — 双重否定 + 疑问 = 最安全的反驳形式
えー、いや、そんなにまずくなくない？
STEP 4 — 包装阶段
疑问形式（なくない？）已内含降级功能
跳过 —「まずくなくない？」本身就是「我不是在断言，是在问你」
STEP 5 — 出口阶段
需要继续解释立场但不想对立
追加个人经验作为缓冲理由 — 不说「你错了」而是「我的经验不一样」
えー、いや、そんなにまずくなくない？私この前行った時は普通に美味しかったけどな
STEP 6 — 实时自检
回顾整句 —「普通に美味しかった」太确定了，有风险
追加降级：把分歧归因于客观因素
...たぶん日によるのかも
最终输出
えー、いや、そんなにまずくなくない？
私この前行った時は普通に美味しかったけどな...
たぶん日によるのかも

替代路径对比

低包装版（亲密朋友）
え、まじ？私あそこ好きだけどな
即时反应(え) → 语义(好き) → 出口悬挂(けどな) ｜ 跳过：压缩、降级
高包装版（不太熟的人）
あー...そうなんだ。なんか私が行った時はそんなに悪くなかった気がするんだけど...好みの問題かもね
反应(あー) → 假装接受(そうなんだ) → 缓冲(なんか) → 压缩(そんなに) → 语义(悪くなかった) → 降级(気がする) → 出口(んだけど) → 社交保险(好みの問題かも)
零包装版（吵架模式）
は？普通に美味しいでしょあの店
反应(は？) → 语义(美味しい) → 出口断言(でしょ) ｜ 跳过：缓冲、压缩、降级全部跳过
05

运行规则

10条核心规则
R01
社交风险决定管线长度
风险越高，启用的阶段越多。风险=0时管线可以只有语义+出口两个阶段
R02
确信度和社交风险冲突时，社交风险优先
即使100%确定对方错了，如果社交风险高，仍然会降级。日语口语中「正确」不如「关系安全」重要
R03
紧迫度可以覆盖一切
当紧迫度=10，所有包装层被跳过。「危ない！」不会变成「なんかちょっと危ない気がするんだけど」
R04
管线可以在任意阶段倒回
说到一半发现风险评估错了，可以中断、道歉、从头重来。这就是为什么日语口语充满了いや、ていうか、そうじゃなくて
R05
同一阶段可以重复加载
社交风险极高时，缓冲可以叠两三层：なんか...あの...ちょっと。降级也可以叠加
R06
出口选择决定对话走向
んだけど=把球传给对方 ｜ よね=拉对方站自己这边 ｜ ...沉默=我说完了你自己想
R07
修正管线随时可以抢占主管线
一旦检测到问题（自检或对方反应），修正管线立即中断主管线并接管
R08
语义载体在整个管线中占比最小
完整的日语口语句中，实际语义内容通常只有1-2个词。其余全部是调节机制
R09
亲密度反向影响管线长度
越亲密的关系管线越短。跟最好的朋友和跟不太熟的人说话，同一个想法产生完全不同长度的句子
R10
疑问形式可以替代降级阶段
把判断变成问题（〜くない？/〜じゃない？）本身就完成了降级——你不是在断言，是在「问」
